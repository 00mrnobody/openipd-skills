# OpenIPD 快速使用说明（Codex / Kimi CLI / Claude Code）

项目地址：
- https://github.com/00mrnobody/openipd-skills

这份说明只讲最短路径：安装、调用、常用指令。

## 1) 安装（自然语言）

项目地址：
- https://github.com/00mrnobody/openipd-skills

你可以直接对助手说：

- Codex：`帮我把这个项目的 OpenIPD skills 安装到这台机器（~/.codex/skills）`
- Kimi CLI：`帮我把 OpenIPD skills 安装到 ~/.config/agents/skills`
- Claude Code：`帮我把 OpenIPD skills 安装到 ~/.claude/skills`

如果你希望一句话同时安装三端：

- `帮我把 openipd-skills 安装到 Codex、Kimi CLI、Claude Code 三个目录`

## 2) 直接用自然语言调用

安装后，直接在对话里用自然语言 + 角色名即可。

### 启动完整流程（推荐）

```text
/ipd-orchestrator 启动IPD流程：
产品：桌面AI陪伴机器人
目标市场：都市白领
预算：1000万
```

### 调单个角色

```text
/ipd-mktpdt 给我做目标市场、竞品、定价建议
/ipd-se 给我做系统架构和技术风险评估
/ipd-fpdt 给我做ROI和盈亏平衡测算
```

### 指定流程阶段

```text
/ipd-orchestrator 从开发阶段开始，执行TR3/TR4评审
/ipd-orchestrator 查询当前流程状态
```

## 3) 不想看长文档，直接出图

你可以让角色直接输出：
- 渲染图
- 信息图（Notion风格）
- Slides单页视觉

示例：

```text
/ipd-lpdt 把这份PDCP结论直接做成3页slides风格信息图
/ipd-mktpdt 把市场分析输出成Notion版式信息图
/ipd-se 把系统方案输出成架构渲染图
```

## 4) 常见问题

- 没看到角色：重启你的 Agent 工具后再试。
- 提示找不到技能：检查目录是否正确、是否复制了 `skills/ipd-*`。
- 图像生成失败：检查是否配置了 `GEMINI_API_KEY`（只放环境变量，不要写进仓库）。

## 5) 一句话总结

安装技能目录后，你只要用自然语言发 `/ipd-orchestrator` 或 `/ipd-角色名`，就能跑流程、出文档、出信息图和Slides。
